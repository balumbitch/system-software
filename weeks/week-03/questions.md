# Вопросы для самопроверки

1. В чем разница между Forward Proxy (обычный прокси) и Reverse Proxy (обратный прокси)?
Forward Proxy (клиентский прокси):
- Защищает клиента, скрывая его от серверов
- Клиенты внутри сети используют его для доступа во внешний мир
- Пример: корпоративный прокси для выхода в интернет
Reverse Proxy (серверный прокси):
- Защищает серверы, скрывая их от клиентов
- Клиенты из внешнего мира обращаются к нему для доступа к внутренним серверам
- Пример: Nginx как API Gateway перед микросервисами

2. Что такое Upstream в терминологии Nginx?
Upstream — это блок конфигурации Nginx, который определяет группу бэкенд-серверов, между которыми Nginx распределяет нагрузку. Указывает адреса и порты серверов, куда проксируются запросы.

3. Зачем скрывать реальные IP-адреса сервисов от клиента?
- Безопасность: злоумышленники не могут атаковать сервисы напрямую
- Сокрытие архитектуры: клиент не знает внутреннюю структуру
- Гибкость: можно менять IP-адреса и порты сервисов без изменения клиентов
- Балансировка: клиент не привязан к конкретному экземпляру сервиса

4. Какие еще задачи, кроме маршрутизации, может выполнять API Gateway (назовите хотя бы 2)?
- Аутентификация и авторизация: проверка токенов JWT, OAuth
- Rate limiting: ограничение количества запросов от одного клиента
- Кэширование ответов: уменьшение нагрузки на бэкенд
- Логирование и мониторинг: сбор метрик и логов всех запросов
- Трансформация протоколов: преобразование REST в gRPC
- Агрегация ответов: объединение данных от нескольких сервисов

5. Что произойдет, если Gateway упадет? Как можно от этого защититься?
Последствия падения Gateway:
- Все клиенты потеряют доступ ко всем сервисам
- Приложение полностью перестанет работать
Способы защиты:
- Репликация: запуск нескольких экземпляров Gateway
- Балансировщик нагрузки: использование дополнительного балансировщика перед Gateway
- Мониторинг и автоматическое восстановление: системы оркестрации (Kubernetes) перезапустят упавший экземпляр
- Graceful degradation: реализация fallback-механизмов на клиенте
- Резервирование: активный и пассивный Gateway с автоматическим переключением
